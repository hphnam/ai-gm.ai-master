<objective>
## Goal
Land prompt-payload format + Anthropic prompt-cache wins on top of 01-02's section-injection. Find_knowledge tool results render with byte-stable `[Section {id} · {title}]\n{content}\n\n` prefix, deterministic ordering. System prompt + tool defs blocks marked `cache_control: { type: 'ephemeral' }` so repeat-turn calls observe `usage.cache_read_input_tokens > 0`. Recalibrate the v0.1 canned 6-query harness against the new section-injection path; document new minSimilarity threshold and add cache-hit + payload-format probes.

## Purpose
01-02 shipped section-injection at the data layer (RetrievalHit.metadata.sectionId/sectionTitle/sectionTokenCount/sectionTruncated populated), but reserved the prompt-payload byte-format under audit-S7 boundary so 01-03 could own cache-stable design. Without that format + cache_control wiring, the new section-injection path is functionally correct but cost-equivalent to v0.2 (every turn re-pays for system prompt + tool defs serialization). This plan closes the per-turn $0.01–0.02 target documented in PROJECT.md — hitting `cache_read_input_tokens > 0` is what converts hierarchical retrieval from "feels right" to "operationally cheap." Probe-eval recalibration measures whether section content denser-than-chunk shifts the retrieval hit/no-data threshold (CONTEXT.md secondary concern: "0.3-similarity no-data threshold may shift").

## Output
- `apps/api/src/modules/chat/tool-dispatcher.ts` — find_knowledge case wraps each RetrievalHit.content with the byte-stable prefix when sectionId is present; sorts hits by similarity DESC then sectionId ASC for tie-break; falls through unchanged when sectionId is absent (AC-5 carry-over from 01-02).
- `apps/api/src/modules/chat/gm-agent.ts` — `providerOptions.anthropic` extended with `cacheControl` markers on the system block + tool defs block per Anthropic + AI SDK contract (verified at APPLY time via WebFetch of @ai-sdk/anthropic docs — API surface moves).
- `apps/api/src/modules/chat/system-prompt.ts` — `contextualInstructions` split into a stable suffix (CHAT_SYSTEM_PROMPT + mode overlay) and a dynamic head (current_context block, user profile, contacts) so the cache prefix is the static suffix; reorder to put dynamic-then-stable.
- `packages/types/src/section.ts` — `SECTION_PAYLOAD_PREFIX_TEMPLATE = '[Section {sectionId} · {sectionTitle}]'` + `formatSectionPayload({ sectionId, sectionTitle, content }): string` helper. Single source of truth, grep-discoverable.
- `apps/api/scripts/probe-eval.ts` — NEW. Canned 6-query harness: per query asserts retrieval returns ≥1 hit OR documents the no-data path explicitly. Aggregate pass rate ≥60% (matches v0.1 04-03 spec). Logs new threshold candidate based on observed similarity floor across the 6.
- `apps/api/scripts/probe-section.ts` — W24 (cache_read_input_tokens > 0 on a 2-turn repeat-query test against ChatService), W25 (find_knowledge tool result payload byte-stable across two consecutive identical retrievals — assert format prefix + sort key), W26 (deterministic ordering — same 5-section corpus, two retrievals with same query, identical sectionId order).
- `apps/api/package.json` — `probe:eval` npm script.
- `01-03-SUMMARY.md` with per-AC outcomes, threshold-recalibration table, cost-log table.
