<objective>
## Goal
Ship the Phase 2 manager self-service document feature: multipart `POST /docs/upload` endpoint accepting .md/.txt/.pdf/.docx files with MIME + size validation, server-side text extraction (unpdf for PDF, mammoth for DOCX, pass-through for text), and `DELETE /docs/:id` with cascade cleanup. Wire the existing `/docs` UI to accept PDF/DOCX via the new upload endpoint and expose a delete button. Probe coverage for upload happy path, MIME/size rejects, DELETE, and cross-org DELETE denial.

## Purpose
Phase 2's headline feature is managers uploading their venue SOPs, checklists, supplier notes directly — not through the existing plain-text paste form. The in-place UI reads .md/.txt client-side but cannot handle PDF/DOCX (binary). Adding a true server-side multipart endpoint with extraction unlocks the realistic content distribution a hospitality manager will have (PDFs from suppliers, DOCX from H&S, spreadsheets-exported-to-docx from training decks). DELETE closes the CRUD loop — currently managers can create but cannot remove. Plan 02-01 closed the cross-org leak; this plan safely rides on that boundary.

## Output
- `multer` + `unpdf` + `mammoth` dependencies added to apps/api (+ `@types/multer`).
- New `apps/api/src/modules/docs/doc-extract.ts` — pure extraction router over four MIME types, returns string text + `ExtractError` for unsupported/corrupt files.
- `POST /docs/upload` endpoint on `DocsController`: multipart, manager-only, 10 MB size cap, MIME allowlist (`text/plain`, `text/markdown`, `application/pdf`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`), routes extracted text through existing `DocsService.create` → `IngestService.ingest`.
- `DELETE /docs/:id` on `DocsController`: manager-only, org-scoped (reuses cross-org denial log pattern from 02-01), cascades `ReTagQueueItem` rows whose `knowledgeItemId` matches (Prisma already handles the FK, just a probe check).
- `apps/web/src/components/docs/doc-form.tsx`: file picker accepts `.md,.txt,.pdf,.docx`; for `.md/.txt` keep existing client-side text read; for `.pdf/.docx` switch to multipart upload.
- `apps/web/src/components/docs/doc-list.tsx`: add delete button per row with confirm dialog.
- `apps/web/src/lib/hooks/use-docs.ts`: add `useUploadDoc` + `useDeleteDoc` mutations.
- `packages/types/src/api.ts`: new API error codes `file-too-large`, `unsupported-file-type`, `extraction-failed`.
- `probe-api.ts`: +5 assertions (A38 upload happy path PDF/DOCX stub; A39 MIME reject; A40 size reject; A41 DELETE happy path; A42 cross-org DELETE denial 404).

       export class ExtractError extends Error {
         constructor(public mimeType: string, public reason: 'unsupported-mime' | 'corrupt-bytes' | 'timeout' | 'empty-result', cause?: unknown) {
           super(`extract failed for ${mimeType}: ${reason}`)
           this.name = 'ExtractError'
           if (cause) (this as { cause?: unknown }).cause = cause
         }
       }

       const MAX_EXTRACT_CHARS = 1_000_000 // 1M char cap post-extraction — prevents OOM on hostile PDFs
       // audit-added S1: extraction timeout — prevents CPU-bound DoS on pathological PDFs
       const EXTRACT_TIMEOUT_MS = 30_000

       async function withTimeout<T>(p: Promise<T>, mimeType: string): Promise<T> {
         let timer: NodeJS.Timeout | undefined
         const timeout = new Promise<never>((_, reject) => {
           timer = setTimeout(() => reject(new ExtractError(mimeType, 'timeout')), EXTRACT_TIMEOUT_MS)
         })
         try {
           return await Promise.race([p, timeout])
         } finally {
           if (timer) clearTimeout(timer)
         }
       }

       export async function extractText(buffer: Buffer, mimeType: string): Promise<string> {
         try {
           let text: string
           switch (mimeType) {
             case 'text/plain':
             case 'text/markdown':
               text = buffer.toString('utf-8').slice(0, MAX_EXTRACT_CHARS)
               break
             case 'application/pdf': {
               const { extractText: unpdfExtract } = await import('unpdf')
               const { text: unpdfText } = await withTimeout(
                 unpdfExtract(new Uint8Array(buffer), { mergePages: true }),
                 mimeType,
               )
               const joined = Array.isArray(unpdfText) ? unpdfText.join('\n\n') : unpdfText
               text = (joined ?? '').slice(0, MAX_EXTRACT_CHARS)
               break
             }
             case 'application/vnd.openxmlformats-officedocument.wordprocessingml.document': {
               const mammoth = await import('mammoth')
               const { value } = await withTimeout(
                 mammoth.extractRawText({ buffer }),
                 mimeType,
               )
               text = (value ?? '').slice(0, MAX_EXTRACT_CHARS)
               break
             }
             default:
               throw new ExtractError(mimeType, 'unsupported-mime')
           }
           // audit-added M3: empty / whitespace-only extraction is not a useful KnowledgeItem —
           // rejecting here prevents polluting the DB and triggering garbage enrichment downstream.
           if (text.trim().length === 0) {
             throw new ExtractError(mimeType, 'empty-result')
           }
           return text
         } catch (err) {
           if (err instanceof ExtractError) throw err
           throw new ExtractError(mimeType, 'corrupt-bytes', err)
         }
       }

       // audit-added M2: title sanitization — strip path separators, control chars, length cap.
       // Prevents filenames like "../../etc/passwd.pdf" or "\x00evil.pdf" landing in user-visible title.
       const TITLE_MAX = 200
       export function sanitizeUploadTitle(originalname: string): string {
         const withoutExt = originalname.replace(/\.(pdf|docx|md|txt)$/i, '')
         const noSeparators = withoutExt.replace(/[\\/]/g, ' ')
         // eslint-disable-next-line no-control-regex
         const noControl = noSeparators.replace(/[\x00-\x1f\x7f]/g, '')
         const trimmed = noControl.trim().slice(0, TITLE_MAX)
         return trimmed.length > 0 ? trimmed : 'Untitled upload'
       }

       export const UPLOAD_MIME_ALLOWLIST = [
         'text/plain',
         'text/markdown',
         'application/pdf',
         'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
       ] as const
       export const UPLOAD_MAX_BYTES = 10 * 1024 * 1024 // 10 MB

## SCOPE LIMITS
- NO MIME sniffing beyond the `Content-Type` header — we trust multer's parsed value. A hostile client can lie about MIME but extraction will throw for mismatched buffer shapes, surfacing as 422 extraction-failed.
- NO virus scanning / ClamAV — post-POC concern.
- NO file history / versioning — single-revision upsert semantics from IngestService apply.
- NO thumbnail / preview generation — text-extraction only.
- NO S3 / object storage — files live ephemerally in multer buffer, only the extracted text persists to DB.
- NO multi-file upload — one file per request.
- NO bulk DELETE — one id per request.
