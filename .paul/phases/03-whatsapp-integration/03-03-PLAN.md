## AC-14: Magic-byte signature validation post-download (audit-added M3)
```gherkin
Given downloadTwilioMedia has fetched bytes for a declared `Content-Type: image/jpeg`
  And the first 3 bytes are NOT `FF D8 FF` (JPEG magic)
When magic-byte validation runs
Then downloadTwilioMedia returns `{ ok: false, reason: 'media-content-mismatch' }`
  And handleInbound emits `whatsapp.image_download_failed { errorKind: 'media-content-mismatch' }`
  And the adapter sends the friendly fallback
  And ChatService is NOT invoked

Magic-byte signatures enforced:
  - JPEG: bytes [0..2] = FF D8 FF
  - PNG:  bytes [0..3] = 89 50 4E 47
  - GIF:  bytes [0..3] = 47 49 46 38 (GIF8)
  - WebP: bytes [0..3] = 52 49 46 46 (RIFF) AND bytes [8..11] = 57 45 42 50 (WEBP)
```
