<objective>
## Goal
Replace the Twilio Verify SMS OTP integration (Phase 1 Plan 01-03) with Infobip 2FA. Same functional surface: `startVerification(phoneNumber)` + `checkVerification(phoneNumber, code)` + `mode` getter (`console` / `live` / `disabled` with runtime kill-switch). Different wire: `POST /2fa/2/pin` (send) + `POST /2fa/2/pin/{pinId}/verify` (check) against the Infobip base URL, with `Authorization: App {INFOBIP_API_KEY}` auth (same header pattern as Plan 03-04's WhatsApp adapter).

Delete `twilio-verify.service.ts`. Drop the `twilio` block from `assertAuthEnv()` — SMS OTP was the last Twilio integration. Clean up historical Twilio comments in WhatsApp/chat modules (past-tense phrasing; no identifier renames).

## Purpose
Plan 03-04 migrated **WhatsApp** from Twilio to Infobip. SMS OTP (Phase 1 Plan 01-03) was explicitly preserved on Twilio because WhatsApp and 2FA are separate Twilio integrations and only the WhatsApp one was in 03-04 scope. Result: the project still carries a Twilio dependency for a single code path.

Ryan's operational decision (2026-04-20 21:28): remove Twilio entirely. `.env` already has all Twilio vars deleted — runtime currently falls back to console mode on every `/auth/phone/send` (confirmed at 21:27:55: `TwilioVerifyService phone.console_fallback code: PROBE-481620`). This plan replaces the service with an Infobip 2FA driver so live mode becomes reachable again.

Infobip 2FA is a managed OTP product (closest 1:1 replacement for Twilio Verify): Infobip owns PIN generation, expiry, attempt counting, and delivery. The app passes a `phoneNumber` and receives a `pinId`; verification passes the `pinId` + user-entered code back to Infobip which responds `verified: true | false`. Application + message-template setup lives in the Infobip Portal — out-of-scope here, documented in the UAT runbook.

## Output
- Provider-specific name on the wire-layer file (`infobip-verify.service.ts`) — mirrors 03-04's adapter+guard naming discipline.
- Public shape unchanged: controller still injects a "phone-verify service" with `startVerification` / `checkVerification` / `mode`. Inside the controller the identifier is renamed `this.twilio` → `this.verify` (provider-agnostic); all three callsites preserved.
- Driver-override env renamed `TWILIO_DRIVER_OVERRIDE` → `PHONE_VERIFY_DRIVER_OVERRIDE` (concern-scoped; separate from `INFOBIP_DRIVER_OVERRIDE` which is WhatsApp-scoped in Plan 03-04).
- `assertAuthEnv()` gains an Infobip-2FA env block (`INFOBIP_2FA_APPLICATION_ID`, `INFOBIP_2FA_MESSAGE_ID`) with all-or-nothing validation. Twilio SMS block removed.
- `.env.example` cleanup: delete orphan `TWILIO_*` entries, add the two new 2FA vars with sourced comments.
- Historical Twilio comments in `whatsapp.adapter.ts:6`, `whatsapp.adapter.ts:23`, `whatsapp.service.ts:341`, `typing-indicator-timers.ts:59`, `chat.service.ts` (if any) updated to past tense — provenance retained, present-tense references removed.
