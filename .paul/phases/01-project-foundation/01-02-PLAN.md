<objective>
## Goal
Define the full Prisma schema (9 models from PAUL.md §4.2) with pgvector-backed embedding columns, run the initial migration against a real Postgres+pgvector database, and export a typed `PrismaClient` singleton from `@gm-ai/database` that every future phase can import.

## Purpose
Every subsequent phase depends on this data model:
- Phase 2 (Embeddings & Seeding) writes rows into Venue/Supplier/StockItem/SopDocument and populates the `vector(1024)` columns via `$executeRaw`.
- Phase 3 (Retrieval) reads those rows via cosine-similarity `$queryRaw`.
- Phase 4 (Chat Engine) persists conversation/message rows with retrieved-ID arrays.
- Phase 5 (Web Interface) consumes the same types transitively.

Without a migrated schema, nothing downstream compiles or runs.

## Output
- `packages/database/prisma/schema.prisma` — canonical data model
- `packages/database/prisma/migrations/<timestamp>_init/migration.sql` — first migration, including `CREATE EXTENSION IF NOT EXISTS vector`
- `packages/database/src/index.ts` — exports `prisma` singleton + re-exports Prisma types
- Root `package.json` updated with `pnpm.onlyBuiltDependencies` for `prisma`, `@prisma/client`, `@prisma/engines`, `@nestjs/core` (unblocks postinstall scripts without interactive `pnpm approve-builds`)
</objective>

    ```ts
    import { PrismaClient } from '@prisma/client'

    const globalForPrisma = globalThis as unknown as { prisma?: PrismaClient }
    export const prisma = globalForPrisma.prisma ?? new PrismaClient()
    if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma

    ```
    The singleton pattern prevents connection exhaustion in Next.js dev hot-reload scenarios later.
