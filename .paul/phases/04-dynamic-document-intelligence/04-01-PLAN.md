<objective>
## Goal
Extend the `/docs` upload pipeline to extract text from five additional formats — **.xlsx, .csv, .pptx, .jpg/.png/.heic/.webp** — while keeping the existing PDF / DOCX / TXT / MD paths byte-identical. Extraction output flows into the existing `KnowledgeItem` embedding + retrieval pipeline unchanged. No classifier, no taxonomy, no procedural detection — those land in Plans 04-02+.

## Purpose
Phase 4 "Dynamic Document Intelligence" treats every upload as input to a per-tenant classifier (Plan 04-02). That classifier can only work if the system can actually *read* the doc. Today's extraction pipeline handles prose formats only; hospitality SMB document reality is spreadsheets, slide decks, and phone photos of laminated sheets. 04-01 removes the format bottleneck so the canary docs (three beerhall xlsx checklists in `docs/`) can be ingested end-to-end before the classifier ships.

Image-via-Claude-vision is near-free because Phase 3 Plan 03-03 already built the multimodal infra (magic-byte guard, streaming byte counter, Claude image content blocks). We reuse those primitives rather than re-implementing.

## Output
- Five new extractor cases in the `doc-extract.ts` dispatcher, each in its own file under `apps/api/src/modules/docs/extractors/`.
- Extended `UPLOAD_MIME_ALLOWLIST` + `sanitizeUploadTitle` extension regex.
- Per-MIME size cap map replacing the single `UPLOAD_MAX_BYTES` constant (XLSX/CSV/DOCX/PDF/MD/TXT → 10MB, PPTX → 15MB, images → 5MB).
- `apps/web/src/app/docs/docs-body.tsx` file picker `accept` attr extended + "supported formats" help text.
- `docs.image_extract_cost` structured log line on every image-via-vision extraction (input tokens + output tokens + estimated USD + mime + imageBytes).
- `docs.image_extract_queued` structured log when a Claude-vision call enters the concurrency queue (audit-S2).
- **<!-- audit-S3/S4 COMMITTED --> Image-bytes persistence:** `KnowledgeItem.sourceImageBytes Bytes?` + `KnowledgeItem.sourceImageMime String?` (Option A, committed — no APPLY-time Option A vs B decision). Additive nullable columns; safe migration against existing rows.
- **<!-- audit-S3 COMMITTED --> CSV library:** `csv-parse/sync` (mature, smaller dep surface than exceljs.csv for single-use-case).
- **<!-- audit-M2 --> Shared `sanitiseError` util** at `apps/api/src/common/sanitise-error.ts`, factored from Phase 3 Plan 03-05 `InfobipVerifyService` impl; consumed by both InfobipVerifyService (retained behavior) and new image extractor.
- `mime` field added to existing `docs.uploaded` structured log (audit-S5) for per-format regression slicing.
- No changes to retrieval, embedding, audit-log shape beyond the mime field, or WhatsApp runtime.
</objective>

         let inFlight = 0
         const queue: Array<() => void> = []
         async function acquireSlot(logger: Logger): Promise<void> {

             return
           }
