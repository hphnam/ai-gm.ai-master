<objective>
## Goal
Ship the core of Phase 4: a `ChatService` that takes `(conversationId, venueId, userMessage)` and produces a Claude-authored reply by running an **Anthropic tool-use loop** over the five tools from Phase 3 (`find_knowledge`, `get_stock_below_par`, `get_stock_by_name`, `get_supplier_by_name`, `get_upcoming_cutoffs`). The loop calls Claude with tools, executes any tool_use blocks by dispatching to `RetrievalService` / `MockOpsService`, feeds the `ToolResult<T>` envelopes back as `tool_result` content blocks, and iterates until Claude emits final text. Every conversation turn is persisted to `chat_conversations` + `chat_messages`, with the user message and assistant reply stored as rows and the assistant row carrying `retrievedItemIds` (from any `find_knowledge` calls) + `toolCallLog` (full `[{tool, input, result}]` trace).

The system prompt enforces the "honest no-data" contract: when a tool returns `{ ok: false, reason: 'no-data' }`, Claude must say so plainly — never invent an answer, never paper over the gap.

## Purpose
CONTEXT.md's "investor-ready demo quality bar" requires three things working end-to-end: an agentic KB (shipped in 03-01/03-02), tool-backed ops access (shipped in 03-03), and a chat engine that uses both honestly. Plans 04-02 (proactive suggestions) and 04-03 (adaptation loop) both layer onto a working chat loop — a below-par suggestion is emitted mid-turn by the same tool machinery this plan defines; a thumbs-down-driven re-tag triggers by replaying a conversation through a retrieval-capturing wrapper.

Equally important: **the audit trail**. Without `retrievedItemIds` + `toolCallLog` persisted per assistant message, there's no way for Plan 04-03's eval harness to ask "did the AI retrieve the right docs for this query?" and no way for Plan 05-02's debug panel to show tool call traces. The persistence piece is load-bearing, not cosmetic.

## Output
- `@gm-ai/types` — `chat-tools.ts` with the canonical Anthropic tool JSON array (`ToolDefinitions`) + Zod input schemas per tool, both exported from dist so `apps/api` imports them and any future consumer (a lightweight external client, Plan 05 UI's type hints) reuses the same shapes.
- `ToolDispatcher` in `apps/api/src/modules/chat/` — thin router from `(tool_name, input)` → `ToolResult<T>` by calling the right `RetrievalService` / `MockOpsService` method. Zod-validates inputs before dispatch; unknown tool names return `fail('not-supported', ...)`.
- `ChatService` in the same module — `sendMessage({ conversationId?, venueId, userMessage })` runs the full tool-use loop (max 6 rounds — audit-capped to prevent infinite loops), persists both messages, returns `{ conversationId, assistantMessage, toolCallLog, retrievedItemIds }`.
- `system-prompt.ts` — single source of truth for the chat system prompt (no-data honesty, ops-data-comes-from-tools, cite retrieved doc summaries verbatim where possible).
- `ChatModule` wired into `AppModule`; imports `RetrievalModule` (which re-exports MockOpsModule), so Phase 3's services arrive as a single import.
- `probe-chat.ts` — runs 3 real end-to-end conversations: (a) a retrieval-backed SOP question, (b) a mock-ops-backed stock-below-par query, (c) a deliberate nonsense question to prove honest no-data surfaces in the reply. Asserts persistence (conversation row, two message rows, correct retrievedItemIds + toolCallLog). Cleans up probe conversations in `try/finally`.
- `pnpm -w build` clean; `pnpm --filter api probe:chat` exits 0; probe:seed + probe:ingest + probe:retrieval all still green.

    ```ts
    import { Injectable } from '@nestjs/common'
    import { TOOL_INPUT_SCHEMAS, type ToolName, fail, type ToolResult } from '@gm-ai/types'
    import { RetrievalService } from '../retrieval/retrieval.service'
    import { MockOpsService } from '../mock-ops/mock-ops.service'

    @Injectable()
    export class ToolDispatcher {
      constructor(
        private readonly retrieval: RetrievalService,
        private readonly mockOps: MockOpsService,
      ) {}

      async dispatch(toolName: string, input: unknown): Promise<ToolResult<unknown>> {
        if (!(toolName in TOOL_INPUT_SCHEMAS)) {
          return fail('not-supported', `tool: ${toolName}`)
        }
        const schema = TOOL_INPUT_SCHEMAS[toolName as ToolName]
        const parsed = schema.safeParse(input)
        if (!parsed.success) {
          return fail('error', `invalid input for ${toolName}: ${parsed.error.issues[0]?.message ?? 'zod error'}`)
        }
        try {
          switch (toolName as ToolName) {
            case 'find_knowledge': {
              const i = parsed.data as { query: string; venueId?: string; limit?: number; minSimilarity?: number }
              return await this.retrieval.find(i.query, {
                venueId: i.venueId,
                limit: i.limit,
                minSimilarity: i.minSimilarity,
              })
            }
            case 'get_stock_below_par':
              return await this.mockOps.getStockBelowPar((parsed.data as { venueId: string }).venueId)
            case 'get_stock_by_name': {
              const i = parsed.data as { venueId: string; name: string }
              return await this.mockOps.getStockByName(i.venueId, i.name)
            }
            case 'get_supplier_by_name':
              return await this.mockOps.getSupplierByName((parsed.data as { name: string }).name)
            case 'get_upcoming_cutoffs': {
              const i = parsed.data as { venueId: string; withinHours?: number }
              return await this.mockOps.getUpcomingCutoffs(i.venueId, i.withinHours)
            }
          }
        } catch (err) {
          const message = (err as Error).message ?? 'unknown dispatcher error'
          // <!-- audit-added --> Structured error log for post-incident forensics.
          // toolCallLog persists the fail envelope; this log gives us the stack-trace-adjacent signal.
          this.logger.error(
            JSON.stringify({ event: 'tool_dispatch.error', tool: toolName, message }),
          )
          return fail('error', message)
        }
      }

      // <!-- audit-added -->
      private readonly logger = new Logger(ToolDispatcher.name)
    }
    ```

      conversationId: string
      assistantMessage: { id: string; content: string }
      toolCallLog: ToolCallLogEntry[]
      retrievedItemIds: string[]
    }

    type AnthropicMsg = Anthropic.Messages.MessageParam

    @Injectable()
    export class ChatService implements OnModuleInit {
      private readonly logger = new Logger(ChatService.name)
      private client!: Anthropic

      constructor(private readonly dispatcher: ToolDispatcher) {}

      onModuleInit() {
        const apiKey = process.env.ANTHROPIC_API_KEY
        if (!apiKey) throw new Error('ANTHROPIC_API_KEY is not set')
        this.client = new Anthropic({ apiKey })
      }

      async sendMessage(rawInput: SendMessageInput): Promise<SendMessageResult> {
        // <!-- audit-added --> 0. Validate input at the trust boundary — no DB write, no Anthropic spend on bad input.
        const parsed = SendMessageInputSchema.safeParse(rawInput)
        if (!parsed.success) {
          throw new Error(
            `invalid sendMessage input: ${parsed.error.issues[0]?.message ?? 'zod error'}`,
          )
        }
        const input = parsed.data

        // <!-- audit-added --> 0a. Venue preflight — verify venueId exists, load name for contextual system prompt.
        const venue = await prisma.venue.findUnique({
          where: { id: input.venueId },
          select: { id: true, name: true },
        })
        if (!venue) throw new Error(`venue ${input.venueId} not found`)

        // <!-- audit-added --> 0b. Conversation preflight — if id provided, verify it exists AND belongs to this venue.
        if (input.conversationId) {
          const existing = await prisma.chatConversation.findUnique({
            where: { id: input.conversationId },
            select: { id: true, venueId: true },
          })
          if (!existing) throw new Error(`conversation ${input.conversationId} not found`)
          if (existing.venueId !== venue.id) {
            throw new Error(
              `conversation ${input.conversationId} does not belong to venue ${venue.id}`,
            )
          }
        }

        // 1. Ensure conversation exists
        const conversationId =
          input.conversationId ??
          (await prisma.chatConversation.create({
            data: { venueId: input.venueId, channel: 'web' },
            select: { id: true },
          })).id

        // 2. Persist user message
        await prisma.chatMessage.create({
          data: {
            conversationId,
            role: 'user',
            content: input.userMessage,
            retrievedItemIds: [],
            toolCallLog: [],
          },
        })

        // 3. Load full history as Claude messages
        // <!-- audit-added --> Deterministic tiebreak by id — prevents turn-order ambiguity when DB clock collides.
        const history = await prisma.chatMessage.findMany({
          where: { conversationId },
          orderBy: [{ createdAt: 'asc' }, { id: 'asc' }],
          select: { role: true, content: true },
        })
        const messages: AnthropicMsg[] = history.map((m) => ({
          role: m.role === 'assistant' ? 'assistant' : 'user',
          content: m.content,
        }))

        // <!-- audit-added --> 3a. Build per-call contextual system prompt so Claude has venueId without having to ask.
        const contextualSystemPrompt = `${CHAT_SYSTEM_PROMPT}\n\n<current_context>\nvenueId: ${venue.id}\nvenueName: ${venue.name}\n</current_context>`

        // 4. Tool-use loop (wrapped in try/catch — every user message gets a paired assistant outcome)
        const toolCallLog: ToolCallLogEntry[] = []
        const retrievedItemIds = new Set<string>()
        let finalText = ''

        try {
          for (let round = 1; round <= MAX_ROUNDS; round++) {
            const roundStart = Date.now()
            const response = await this.client.messages.create({
              model: 'claude-sonnet-4-6',
              max_tokens: MAX_TOKENS,
              system: contextualSystemPrompt,
              tools: TOOL_DEFINITIONS as unknown as Anthropic.Messages.Tool[],
              messages,
            })

            // <!-- audit-added --> Per-round structured observability — no PII, metadata-only.
            this.logger.log(
              JSON.stringify({
                event: 'chat.claude_call',
                conversationId,
                round,
                stop_reason: response.stop_reason,
                input_tokens: response.usage?.input_tokens ?? null,
                output_tokens: response.usage?.output_tokens ?? null,
                latency_ms: Date.now() - roundStart,
              }),
            )

            // Append Claude's assistant turn to history for the next round
            messages.push({ role: 'assistant', content: response.content })

            if (response.stop_reason !== 'tool_use') {
              const textBlocks = response.content.filter(
                (b): b is Anthropic.Messages.TextBlock => b.type === 'text',
              )
              finalText = textBlocks.map((b) => b.text).join('\n').trim()
              // <!-- audit-added --> Never persist empty-string assistant content.
              if (!finalText) {
                this.logger.warn(
                  JSON.stringify({
                    event: 'chat.empty_assistant_text',
                    conversationId,
                    stop_reason: response.stop_reason,
                  }),
                )
                finalText = "I couldn't produce an answer — please retry or rephrase."
              }
              break
            }

            const toolUseBlocks = response.content.filter(
              (b): b is Anthropic.Messages.ToolUseBlock => b.type === 'tool_use',
            )

            // Execute tools in parallel per round
            const results = await Promise.all(
              toolUseBlocks.map(async (block) => ({
                block,
                result: await this.dispatcher.dispatch(block.name, block.input),
              })),
            )

            for (const { block, result } of results) {
              toolCallLog.push({
                round,
                toolUseId: block.id,
                tool: block.name,
                input: block.input,
                result,
              })
              if (block.name === 'find_knowledge' && result.ok) {
                const hits = result.data as Array<{ id: string }>
                for (const h of hits) retrievedItemIds.add(h.id)
              }
            }

            // Feed tool_result blocks back as the next user message
            messages.push({
              role: 'user',
              content: results.map(({ block, result }) => ({
                type: 'tool_result',
                tool_use_id: block.id,
                content: JSON.stringify(result),
                is_error: !result.ok && result.reason === 'error',
              })) as unknown as Anthropic.Messages.ToolResultBlockParam[],
            })

            if (round === MAX_ROUNDS) {
              this.logger.warn(
                JSON.stringify({ event: 'chat.tool_loop_capped', conversationId, rounds: MAX_ROUNDS }),
              )
              finalText =
                'I hit the tool-use round limit while working on your question — can you narrow it down?'
            }
          }
        } catch (err) {
          // <!-- audit-added --> Anthropic API error (rate limit, 5xx, network, timeout). Persist partial trail.
          const message = (err as Error).message ?? 'unknown anthropic error'
          this.logger.error(
            JSON.stringify({
              event: 'chat.anthropic_error',
              conversationId,
              rounds_completed: toolCallLog.length > 0
                ? Math.max(...toolCallLog.map((e) => e.round))
                : 0,
              message,
            }),
          )
          finalText = 'I hit an error calling the model — please retry.'
        }

        // 5. Persist assistant message (ALWAYS — even on error paths, so audit trail is complete)
        const assistantMessage = await prisma.chatMessage.create({
          data: {
            conversationId,
            role: 'assistant',
            content: finalText,
            retrievedItemIds: Array.from(retrievedItemIds),
            toolCallLog: toolCallLog as unknown as object,
          },
          select: { id: true, content: true },
        })

        return {
          conversationId,
          assistantMessage,
          toolCallLog,
          retrievedItemIds: Array.from(retrievedItemIds),
        }
      }
    }
