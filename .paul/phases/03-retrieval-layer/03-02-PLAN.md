<objective>
## Goal
Turn the ad-hoc seed-time enrichment into a real agentic ingest pipeline: a `IngestService` that takes raw content + (optional) title / category / venueId, asks Claude to author **freeform** metadata (summary, tags, docType, crossRefs, *plus any emergent keys Claude finds useful*), validates the shape with a soft Zod passthrough schema from `@gm-ai/types`, resolves cross-references to existing knowledge items by title (best-effort), composes embedding text, embeds via Voyage, and persists the row + vector in a single transaction.

## Purpose
The CONTEXT.md vision requires that "when a doc is added, Claude classifies, tags, and cross-references it. No predefined taxonomy." Plan 03-01 shipped the schema + a `{ docType, tags, category }` starter inside the seeder. That's the scaffolding. Plan 03-02 is the actual agentic ingest: emergent metadata keys allowed (`crossRefs`, `contactNames`, `errorCodes`, `timeOfDay`, whatever Claude thinks matters), documented soft contract in `@gm-ai/types`, and a reusable service the seeder AND future ingest paths (CLI, REST endpoint in Plan 04+, drag-drop UI in Phase 5) all call the same way.

Without this, Plan 03-03 retrieval has nothing richer than starter tags to match against; and Plan 04-03 adaptation loop has no path to re-run ingest on a doc (re-tag = re-ingest).

## Output
- `@gm-ai/types` — `KnowledgeMetadata` Zod schema with `passthrough()`; known optional fields (`summary`, `tags`, `docType`, `category`, `crossRefs`); emergent keys allowed; published to dist so runtime imports work (same pattern as `@gm-ai/database` post-02-02).
- `IngestService` (NestJS) in `apps/api/src/modules/ingest/` — `ingest(input)` method that runs the full Claude-enrich → validate → resolve-crossRefs → embed → persist flow. Shares the fence-stripper + 1-retry + fail-soft pattern established in 02-02 / 03-01.
- `SeedCommand` refactored to call `IngestService.ingest()` instead of `EnrichmentService.enrichKnowledgeDoc()` + manual embedding + raw-SQL update. Deletes `enrichment.service.ts` (behaviour absorbed into IngestService).
- `probe-ingest.ts` — runs `IngestService.ingest()` on one fresh ad-hoc doc (not in the seed fixture), verifies agentic metadata + emergent keys + cross-ref resolution + embedding storage, then cleans up the probe row. Exposed as `pnpm --filter api probe:ingest`.
- `pnpm -w build` clean; `pnpm seed` still idempotent; `pnpm probe:seed` + `pnpm probe:ingest` both exit 0.
</objective>

    import { NestFactory } from '@nestjs/core'
    import { IngestModule } from '../src/modules/ingest/ingest.module'
    import { IngestService } from '../src/modules/ingest/ingest.service'
    import { prisma } from '@gm-ai/database'

    const PROBE_ID = 'f0000000-0000-4000-8000-000000000001'
    const STARTER_KEYS = new Set(['summary', 'tags', 'docType', 'category', 'crossRefs'])

    async function runProbe(): Promise<boolean> {
      // audit-added: bootstrap IngestModule directly, not AppModule — narrower scope, faster startup, fewer unrelated side effects.
      const app = await NestFactory.createApplicationContext(IngestModule, { logger: false })
      const ingest = app.get(IngestService)

      await prisma.knowledgeItem.deleteMany({ where: { id: PROBE_ID } })

      const input = {
        id: PROBE_ID,
        title: 'Friday Night Menu Pairing Notes',
        category: 'menu',
        content: `These are unofficial pairing suggestions for the Friday pub-quiz crowd at The Crown.

        When guests order the steak special, recommend the Merlot (house red) — full-bodied, cuts through the fat. Do not recommend the Pinot Grigio with steak.

        For the fish and chips, steer toward a Neck Oil Session IPA or an Aspall Cider if they prefer not ale. Avoid heavy red wines.

        Vegetarian guests asking for pairings: Pinot Grigio (house white) with the halloumi skewers; Prosecco if they're celebrating.

        Bar staff: if a guest asks and you're unsure, refer to the weekly ordering supplier list or ask Sarah Mitchell (Head Bartender).`,
        venueId: null,
      }

      // audit-added: retry once if the first ingest hits fail-soft (empty tags + null aiSummary).
      // One-shot Claude flakes shouldn't red the probe; genuine regressions will fail twice.
      let result = await ingest.ingest(input)
      let md0 = (await prisma.knowledgeItem.findUnique({ where: { id: PROBE_ID }, select: { metadata: true } }))?.metadata as Record<string, unknown> | null
      const failSoftDetected =
        (!md0 || !Array.isArray(md0?.tags) || (md0.tags as unknown[]).length === 0) &&
        result.aiSummary === null
      if (failSoftDetected) {
        console.warn('WARN: first ingest hit fail-soft — retrying once before failing probe')
        await prisma.knowledgeItem.deleteMany({ where: { id: PROBE_ID } })
        result = await ingest.ingest(input)
      }

      const row = await prisma.knowledgeItem.findUnique({
        where: { id: PROBE_ID },
        select: { id: true, metadata: true, aiSummary: true },
      })
      const embeddedCount = await prisma.$queryRawUnsafe<{ n: bigint }[]>(
        `SELECT COUNT(*)::bigint AS n FROM "knowledge_items" WHERE id = $1 AND embedding IS NOT NULL`,
        PROBE_ID,
      )
      const hasEmbedding = Number(embeddedCount[0].n) === 1

      const md = (row?.metadata ?? {}) as Record<string, unknown>
