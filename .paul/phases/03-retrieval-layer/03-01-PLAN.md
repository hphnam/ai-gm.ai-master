<objective>
## Goal
Reshape the database to match the agentic-KB vision from the Phase 3 discussion: drop the rigid `SopDocument` model, introduce a shapeless `KnowledgeItem` model with freeform `metadata Json`, rename ops-data tables to `mock_*` to make the "not yet integrated with Xero/Square" state visible in schema and code, and re-seed the fixture against the new shape.

## Purpose
Everything in Phases 3–5 (agentic ingest, tool-based ops access, proactive suggestions, adaptation loop) depends on the schema being type-free on the knowledge side and honestly-mocked on the ops side. No downstream plan can be built against the old `SopDocument` / `StockItem` shape without re-introducing the hardcoded document-type assumption the user rejected.

## Output
- `KnowledgeItem` model replacing `SopDocument` — `content` + `metadata Json` + `embedding vector(1024)` + `aiSummary`. No `category` enum, no `aiTags` column (tags live inside `metadata`).
- Ops models renamed to `MockStock`, `MockSupplier`, `MockPurchaseOrder`, `MockPurchaseOrderItem`, `MockStockCategory` — mapped to `mock_*` table names via `@@map`. Schema comment at the top of each mock block: "TEMPORARY — replaced by Xero/Square integration in a later milestone."
- `ChatMessage.retrievedSopIds` + `retrievedStockIds` replaced by `retrievedItemIds String[]` + `toolCallLog Json` to stay shape-free as tools expand in Plan 04-01.
- Migration generated and applied to NeonDB (destructive: drops old tables, creates new — seed data is fixture, not prod).
- Seeder rewritten to populate the new tables; SOPs land in `KnowledgeItem` with starter `metadata = { docType, tags, category }` filled by the existing Claude enrichment. Full agentic metadata comes in Plan 03-02.
- `pnpm seed` runs clean and idempotent; `pnpm probe:seed` verifies coverage against the new shape.
</objective>
