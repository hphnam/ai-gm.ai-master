<objective>
## Goal
Land the chat-v2 architecture skeleton end-to-end for **lookup mode only**, behind a per-org feature flag, with cost-capture infrastructure live from day one. Vertical slice that proves the role-based pipeline works before 06-02 expands it.

## Purpose
Phase 6 CONTEXT.md decisions D-06-A through D-06-H lock the architecture; this plan ships the smallest provable slice. Once a real lookup query (e.g. "what's below par?") flows Triage → Docs researcher → Writer (lookup mode) → terse answer with cost recorded — we know the pipeline shape works and 06-02 can layer Analyser/Critic/4 more researchers onto a tested foundation. Cost columns landing in 06-01 means 06-02's expensive pipeline is observable from its first invocation, never blind.

## Output
- New `chat-v2/` NestJS module with Triage + 1 researcher (Docs) + Writer (lookup) + cost tracker
- 2 shaped tools (`get_checklist`, `search_docs`) replacing `find_knowledge` for chat-v2 lookup path
- `Organization.chatV2Enabled` boolean flag; `chat.controller.ts` dispatches to v1 or v2 based on flag
- `ChatMessage.costUsd` + `KnowledgeItem.ingestionCostUsd` Decimal columns + helpers in `apps/api/src/types/cost.ts`
- `writer-examples.ts` seeded with lookup-mode calibration anchors (4 examples from CONTEXT.md)
- `probe-chat-v2.ts` covering lookup mode end-to-end, ≥12 assertions, idempotent across 2 consecutive runs
</objective>
