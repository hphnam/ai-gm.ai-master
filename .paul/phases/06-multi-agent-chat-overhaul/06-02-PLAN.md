<objective>
## Goal
Land pipeline depth: Analyser + Critic + reasoning/incident Writer modes + voice corpus expansion (12 total) + Triage prompt mode-routing. After 06-02, chat-v2 can answer all 3 mode classes (lookup/reasoning/incident) with mode-shaped output, structured re-research circuit-breaking, and Critic-verified specifics on high-stakes turns.

## Purpose
06-01 proved the role-based pipeline shape works for lookup mode. The user pain that drove Phase 6 — generic responses to complex questions, dual-checklist hallucinations, meta-narration leaks — is partly cured (lookup mode) but the harder failure modes (reasoning + incident) still flow through chat-v1 today (because flag dispatcher routes non-lookup correctly to lookup-only chat-v2 in 06-01 by falling through, but chat-v1 is still default-on for everyone). 06-02 makes chat-v2 a complete substitute for chat-v1: every mode covered, voice corpus locked, Critic verifying high-stakes specifics. After 06-02 + 06-03 + 06-04, the cutover gate becomes runnable.

This plan ships **depth before breadth** per CONTEXT.md D-06-02-E re-slicing rationale. The 3-mode Writer + Analyser + Critic stack is more architecturally novel than the 4 researchers (06-03 scope, mostly tool-wrapping). Concentrating risk here means we discover voice/shape problems with the Docs researcher alone before investing 4 more researcher builds.

## Output
- New `analyser.service.ts` (Sonnet 4.6, generateObject with structured output emitting `{synthesis, citations, openQuestions, suggestedShape, evidenceSufficiency}`)
- New `critic.service.ts` (Haiku 4.5, takes Writer draft + citations, returns `{verdict, corrections?}`)
- Writer extended to handle 3 modes (lookup/reasoning/incident); 2 new mode-specific prompts; existing lookup prompt unchanged
- `writer-examples.ts` extends from 4 → 12 examples (4 lookup unchanged + 4 reasoning + 4 incident)
- Triage prompt expanded for mode-routing + boundary edge cases (e.g. flat-pint complaint with allergen mention → reasoning escalates to incident)
- Orchestrator dispatches per-mode; Analyser → Critic loop with re-research circuit-breaker (confidence < 0.6 + cost < $0.05 → second pass with refined brief)
- Stream phase events emitted backend-side (frontend consumption is 06-04)
- `CostBreakdown` extends to include `analyser` + `critic` accumulators
- 5 new constants in types/chat-v2.ts (thresholds + timeouts + STREAM_PHASE_EVENTS enum + types)
- `probe-chat-v2.ts` extends with ~25 new assertions covering reasoning/incident shape, Analyser confidence + re-research dispatch, Critic threshold gating + correction loop, Cost breakdown 5-stage coverage; idempotent across 2 runs; zero regression on existing V1-V19
</objective>

    **prompts/analyser.prompt.ts (NEW):**
    - Slim (~50 lines). Tells Sonnet:
      - Role: "You are the Analyser. You DO NOT write to the user — that's the Writer's job. You read what researchers retrieved, reconcile any overlaps, decide the answer SHAPE, and judge whether the evidence is enough to ship."
      - Inputs: userMessage, mode (reasoning | incident), researcher findings array (each with summary + citations).
      - Tasks:
        1. Synthesize the findings into a single coherent narrative the Writer can use directly. Cite specifics — numbers, names, codes — verbatim from researchers, never paraphrasing them.

        3. Identify openQuestions — what's missing that would change the answer.
        4. Self-rate evidenceSufficiency from 0 to 1: 1 = "I have everything I need to give a confident answer"; 0.5 = "I can give a partial answer but it's thin"; 0 = "I have nothing useful". Return a single number.
