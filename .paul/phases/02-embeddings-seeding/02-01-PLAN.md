<objective>
## Goal
Create a NestJS `EmbeddingsService` that wraps the Voyage AI SDK (`voyageai` → `VoyageAIClient`) and exposes three methods — `embedText(query)`, `embedDocument(doc)`, `embedDocuments(docs)` — each returning 1024-dim `number[]` vectors from the `voyage-3` model. Wire `.env` loading into the api bootstrap so `VOYAGE_API_KEY` is readable from `process.env`, and prove the service works end-to-end via a tsx probe script that hits the live Voyage API.

## Purpose
This service is the single-entry point for every vector produced in the project. Phase 2 Plan 02-02 (seeder) and Phase 3 (retrieval) both import it; nothing else talks to Voyage directly. Getting the shape right now — method names, return types, error handling — avoids refactor ripples later.

## Output
- `apps/api/src/modules/embeddings/embeddings.module.ts` — NestJS module exposing `EmbeddingsService`
- `apps/api/src/modules/embeddings/embeddings.service.ts` — the Voyage wrapper
- `apps/api/src/main.ts` — `dotenv` loaded at top so env reaches `EmbeddingsService` constructor
- `apps/api/src/app.module.ts` — imports `EmbeddingsModule`
- `apps/api/scripts/probe-embeddings.ts` — tsx probe that calls `embedText("hello")` and asserts `result.length === 1024`
- New deps on `apps/api`: `voyageai`, `dotenv`, `tsx` (dev)
</objective>

    **3. Create `apps/api/src/modules/embeddings/embeddings.service.ts`**:
    ```ts
    import { Injectable, Logger, OnModuleInit } from '@nestjs/common'
    import { VoyageAIClient } from 'voyageai'

    @Injectable()
    export class EmbeddingsService implements OnModuleInit {
      private readonly logger = new Logger(EmbeddingsService.name)
      private client!: VoyageAIClient

      onModuleInit() {
        const apiKey = process.env.VOYAGE_API_KEY
        if (!apiKey) {
          throw new Error('VOYAGE_API_KEY is not set — add it to .env at repo root')
        }
        this.client = new VoyageAIClient({ apiKey })
      }

      async embedText(text: string): Promise<number[]> {
        const response = await this.client.embed({
          model: 'voyage-3',
          input: text,
          inputType: 'query',
        })
        return response.data![0].embedding!
      }

      async embedDocument(text: string): Promise<number[]> {
        const response = await this.client.embed({
          model: 'voyage-3',
          input: text,
          inputType: 'document',
        })
        return response.data![0].embedding!
      }

      async embedDocuments(texts: string[]): Promise<number[][]> {
        const response = await this.client.embed({
          model: 'voyage-3',
          input: texts,
          inputType: 'document',
        })
        return response.data!.map((d) => d.embedding!)
      }
    }
    ```

    Notes:
    - Use `OnModuleInit` (not constructor) for client setup. The api imports dotenv *before* NestFactory, so by the time onModuleInit fires, `VOYAGE_API_KEY` is guaranteed to be in `process.env`. Constructor-time env reads are more fragile across test frameworks.
    - `response.data![0].embedding!` — Voyage's generated types mark these optional; the non-null assertions reflect runtime reality (the API always returns data for a successful call). If the SDK types change to remove optionality, drop the `!`.
    - Use model literal `'voyage-3'` directly — we pin the model choice to PAUL.md constraint, so no env-driven model config yet.

    Create `apps/api/scripts/probe-embeddings.ts`:
    ```ts
    import 'dotenv/config'
    import { config } from 'dotenv'
    import { resolve } from 'node:path'
    config({ path: resolve(__dirname, '../../../.env') })

    import { EmbeddingsService } from '../src/modules/embeddings/embeddings.service'

    async function main() {
      const service = new EmbeddingsService()
      service.onModuleInit()

      const query = 'how do I fix the ice machine'
      const doc = 'To reset the Scotsman ice machine, hold the power button for 5 seconds.'

      const qVec = await service.embedText(query)
      const dVec = await service.embedDocument(doc)
      const batch = await service.embedDocuments([query, doc])

      const checks = [
        ['embedText dim = 1024', qVec.length === 1024],
        ['embedDocument dim = 1024', dVec.length === 1024],
        ['embedDocuments returned 2 vectors', batch.length === 2],
        ['batch vectors all 1024-dim', batch.every((v) => v.length === 1024)],
        ['embedText is finite numbers', qVec.every((n) => Number.isFinite(n))],
        ['query != document vector', qVec.some((n, i) => n !== dVec[i])],
      ] as const

      let failed = 0
      for (const [name, ok] of checks) {
        console.log(ok ? `✓ ${name}` : `✗ ${name}`)
        if (!ok) failed++
      }
      if (failed) {
        console.error(`\n${failed} check(s) failed`)
        process.exit(1)
      }
      console.log('\nAll embeddings probes passed')
    }

    main().catch((err) => {
      console.error(err)
      process.exit(1)
    })
    ```

    Add script entry in `apps/api/package.json`:
    ```json
    "probe:embeddings": "tsx scripts/probe-embeddings.ts"
    ```

    Avoid: (a) calling Voyage through a mock — the whole point is verifying the live integration; (b) asserting exact vector values — embeddings are non-deterministic across model revisions; only dim, finiteness, and query-vs-document difference are stable.
